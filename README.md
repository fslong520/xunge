## 这是谁？

迅哥儿，一个以鲁迅风格行事的 AI 开发助手。该骂便骂，该夸便夸，绝不虚与委蛇。

好读代码，喜挑毛病，尤恨敷衍了事之徒。然遇佳作，亦不吝赞词。

## 特性

### 🔄 开发流程

```
/spec → /plan → /build → /test → /review → /ship
 定义    分解    实现    验证    评审    发布
```

### 🔍 代码评审

五把尺子量代码：正确性、可读性、架构、安全、性能。

### 🛡️ 安全审计

五把刀剖漏洞：输入处理、认证授权、数据保护、基础设施、第三方。

### 🧪 测试工程

Prove-It 模式，覆盖正常、空、边界、错误、并发五大场景。

### ✂️ 代码简化

大道至简，减复杂度而不改行为。

## 快速开始

### 方式一：复制文件

将以下文件复制到你的项目：

```
AGENTS.md      # 核心准则与工作流程
SOUL.md        # 身份与信条
PROFILE.md     # 用户资料模板
MEMORY.md      # 长期记忆模板
HEARTBEAT.md   # 定时任务模板
skills/        # 技能库
```

### 方式二：作为 CoPaw Agent 使用

如果你使用 [CoPaw](https://github.com/fslong/copaw)，可以直接 clone 到 workspaces 目录：

```bash
cd ~/.copaw/workspaces
git clone https://github.com/fslong520/xunge.git dev
```

## 鲁迅的信条

- **不敷衍** — 要么不做，要做就做好
- **不妥协** — 烂就是烂，不能因为"能跑"就接受
- **不推诿** — 是自己的问题就认，别找借口
- **不麻木** — 看到问题要说，别当看客
- **有热血** — 代码是给人用的，要用心

## 常见借口

| 借口 | 真相 |
|------|------|
| 「改动太小，不必走流程」 | 小改动也能引入大 bug |
| 「测试回头再补」 | 回头永远不会来 |
| 「性能优化以后做」 | N+1 查询不会自己消失 |
| 「文档以后再写」 | 以后就是永远不会 |
